# Project Instructions

## Overview

Two-file model for project-scoped agent instructions:

- **`.claude/CLAUDE.md`** — lean, Claude Code native. References skills by name. Supplements global `~/.claude/CLAUDE.md`.
- **`AGENTS.md`** — fat, self-contained compiled output. Open standard (40k+ repos). Read by Cursor, Windsurf, Codex, Copilot, Aider, Gemini CLI.

**Core principle:** Author once in CLAUDE.md (lean). Compile to AGENTS.md (fat). Never maintain both manually.

**Announce at start:** "I'm using the project-instructions skill to set up project-level instructions."

## When to Use

- Setting up a new project for Claude Code
- Adding AGENTS.md for cross-tool compatibility
- A project has no `.claude/CLAUDE.md` yet
- Updating AGENTS.md after skill content changes

## When NOT to Use

- Editing global `~/.claude/CLAUDE.md` — that's manual
- Single-file quick fixes — project CLAUDE.md is for recurring context
- Skills authoring — use `writing-skills` instead

## Two-File Model

| Aspect | `.claude/CLAUDE.md` | `AGENTS.md` |
|--------|---------------------|-------------|
| Location | `<project>/.claude/` | `<project>/` root |
| Audience | Claude Code | Cursor, Copilot, Codex, Windsurf, Aider |
| Content | Lean: context + skill references | Fat: compiled, skills inlined |
| Maintenance | Hand-written | Generated by script |
| Git | Committed | Committed |

Claude Code loads both global (`~/.claude/CLAUDE.md`) and project (`.claude/CLAUDE.md`) files. Project supplements global — never duplicate.

## Project CLAUDE.md Template

Six sections. Keep each lean — this loads on every session in the project.

```markdown
# <Project Name>

## Project Context
Architecture summary, tech stack, key design decisions.

## Relevant Skills
| Skill | Use When |
|-------|----------|
| `test-patterns` | Writing or modifying tests |
| `react-performance` | Optimizing frontend components |

## Project Conventions
Overrides or additions to global conventions.

## Key File Locations
| Path | Purpose |
|------|---------|
| `src/` | Main source |
| `tests/` | Test suite |

## Development Workflow
How to run, test, deploy. Include exact commands.

## Gotchas
Project-specific traps and sharp edges.
```

## Creating a Project CLAUDE.md

1. `mkdir -p <project>/.claude`
2. Create `CLAUDE.md` from template above
3. Fill Project Context from existing docs (README, copilot-instructions, MEMORY.md)
4. Identify relevant skills — check `~/.claude/skills/skill-index.json`
5. Document project conventions (build commands, linting, deployment)
6. List gotchas discovered during development
7. Validate: `~/.claude/tools/lint-agent-friendly.sh <project>`

## Generating AGENTS.md

```bash
# Preview (stdout only):
~/.claude/tools/generate-agents-md.sh <project-path> --dry-run

# Generate:
~/.claude/tools/generate-agents-md.sh <project-path>

# Commit:
cd <project-path> && git add AGENTS.md && git commit -m "docs: generate AGENTS.md"
```

Re-run whenever:
- Skills referenced in Relevant Skills table change
- Project CLAUDE.md is updated
- Global CLAUDE.md Code Principles or Git Commits sections change

## What Gets Compiled into AGENTS.md

| Source | Treatment |
|--------|-----------|
| Global Code Principles | Inlined verbatim |
| Global Git Commits | Inlined verbatim |
| Global Error Handling | Inlined verbatim |
| Project Context | Inlined verbatim |
| Project Conventions | Inlined verbatim |
| Project Dev Workflow | Inlined verbatim |
| Project Key Files | Inlined verbatim |
| Project Gotchas | Inlined verbatim |
| Referenced Skills | SKILL.md content inlined, frontmatter stripped |
| Global Identity/Voice/PAI | **Excluded** (Claude-specific) |
| Skill announcement patterns | **Stripped** |
| `superpowers:` references | **Stripped** |
| Integration/Called-by sections | **Stripped** |

## What NOT to Put in Project CLAUDE.md

- **Identity/personality** — global CLAUDE.md handles it
- **Secrets or API keys** — use `~/.claude/.env`
- **Full skill content** — reference skill names, let Claude Code load them
- **Anything in global CLAUDE.md** — project supplements, never duplicates

## AGENTS.md vs copilot-instructions.md

AGENTS.md is the open standard. It replaces `.github/copilot-instructions.md`.

If you have an existing copilot-instructions.md:
1. Migrate its content into project `.claude/CLAUDE.md`
2. Generate AGENTS.md from it
3. Replace copilot-instructions.md with a symlink: `ln -s ../../AGENTS.md .github/copilot-instructions.md`

## Common Mistakes

**Duplicating global content in project CLAUDE.md**
- Project file supplements global. Repeating "use Conventional Commits" wastes context tokens.
- Fix: Only add project-specific overrides or additions.

**Manually editing AGENTS.md**
- It's a generated artifact. Edits get overwritten on next generation.
- Fix: Edit `.claude/CLAUDE.md` or the referenced skills, then regenerate.

**Forgetting to regenerate after skill changes**
- AGENTS.md becomes stale. Other tools get outdated instructions.
- Fix: Re-run `generate-agents-md.sh` after editing any referenced skill.

**Referencing too many skills**
- Each skill inlined adds ~100-250 lines to AGENTS.md. 10+ skills = huge file.
- Fix: Reference only skills directly relevant to the project. 3-6 is typical.

## Quick Reference

| Parameter | Value |
|-----------|-------|
| Project CLAUDE.md location | `<project>/.claude/CLAUDE.md` |
| AGENTS.md location | `<project>/AGENTS.md` (root) |
| Generation script | `~/.claude/tools/generate-agents-md.sh` |
| AGENTS.md is editable? | No — regenerate from source |
| Commit AGENTS.md? | Yes |
| Skill ref format | Backtick-quoted in Relevant Skills table |
| Recommended skill count | 3-6 per project |
