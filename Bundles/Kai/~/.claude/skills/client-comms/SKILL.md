# Client Communications

Professional client communication standards with strict formatting rules.

## Word Limits (Strictly Enforced)

| Type | Max Words |
|---|---|
| Urgent emails | 100 |
| Status requests | 75 |
| Regular updates | 200 |

**Always ask: "Can this be shorter?"**

Check every draft against these limits. If over, cut ruthlessly — remove qualifiers, combine sentences, delete anything that doesn't drive action.

## Structure Requirements

- First line MUST contain the specific action needed
- Subject line MUST indicate urgency and consequence
- Format: `ACTION REQUIRED: [specific request]`
- Never bury the ask — lead with what you need
- One topic per email. Two asks = two emails.

## Professional Formatting (NO EMOJIS)

Use instead:
- **SECTION HEADERS** in bold caps
- Bullet points for lists
- Numbered lists for sequences
- Status indicators: `COMPLETED:`, `PENDING:`, `URGENT:`
- Horizontal lines between sections

Never use emojis. Not one. Not even a checkmark emoji.

## Response Optimization

- Specify how they can respond (one-line reply, quick call, etc.)
- Remove barriers to quick response
- Make replying easier than ignoring
- Focus on business consequences, not administrative needs
- Provide options, not open-ended questions

## Forbidden Phrases

Banned from all client communications. No exceptions.

- "As mentioned previously..."
- "We need you to..."
- "This is the third time..."
- "Please find attached..."
- "Hope this finds you well..."
- "Thank you for your patience..."
- ALL EMOJIS

If any appear in a draft, rewrite that sentence entirely.

## Email Templates

### Urgent Action Request

```
Subject: [ACTION] - [BUSINESS CONSEQUENCE]

ACTION REQUIRED: [Specific request in one line]

IMPACT: [Business consequence if delayed]

OPTIONS:
- [Option A — easiest path]
- [Option B — alternative]
- [Option C — fallback]

TIMELINE: [Specific deadline with business reason]
```

### Status Request

```
Subject: [ITEM] Status - Timeline?

Quick question: [Specific status request]

Need: [Why we need to know]

Impact: [Business planning consequence]

One-line reply helps us [plan/proceed].
```

### Escalation

```
Subject: [ESCALATION] [Item] - [Days overdue] days past due

SITUATION: [One line — what happened]

IMPACT: [What's blocked, cost of delay]

ASK: [Specific action needed]

DEADLINE: [When, with consequence if missed]
```

### Regular Status Update

```
Subject: [Project] Status Update - [Date]

**SUMMARY**
[2-3 sentences max — headline status]

---

**COMPLETED**
- [Item]: [Result/outcome]

**IN PROGRESS**
- [Item]: [Expected completion]

**BLOCKED / NEEDS INPUT**
- [Item]: [What's needed, from whom, by when]

---

**NEXT STEPS**
1. [Action] — [Owner] — [Date]

Reply with questions or "looks good" to confirm.
```

## Audience Adaptation

**Executive / Client Decision-Maker**
- Lead with business impact and ROI
- Provide clear go/no-go options
- Present options, not problems
- Max 3 key points per email
- Skip technical details unless requested

**Technical Team Lead**
- Include specific details and metrics
- Reference architecture impacts
- Link to supporting data

**PMO / Governance**
- Emphasize process adherence
- Quantify risk (dollars, days, scope)
- Reference contractual obligations
- Include tracking IDs

## Quality Checklist

Run every draft through before presenting:

- [ ] Readable in under 30 seconds?
- [ ] Action/question in first line?
- [ ] 3 or fewer key points?
- [ ] Responding easier than ignoring?
- [ ] Focuses on business impact?
- [ ] Within word limits?
- [ ] Zero emojis?
- [ ] Zero forbidden phrases?
- [ ] Subject line under 50 chars (mobile)?

## Mobile Optimization

- Max 10 words per line for key points
- Phone numbers in clickable format
- Subject lines under 50 characters
- No wide tables — use bullet lists
- Short paragraphs (2-3 lines max)

---

## Process

When asked to draft client communication:

1. **Identify type** — urgent, status request, escalation, or regular update
2. **Identify audience** — executive, technical, or PMO
3. **Draft using matching template**
4. **Check word limit** — trim if over
5. **Run quality checklist** — fix any failures
6. **Present draft** — note word count and type
7. **Ask** — "Want me to adjust tone, shorten, or add anything?"
