# Design:A11y Sub-Skill

Deep expertise in WCAG 2.2, A11y Project checklist, and ARIA patterns for accessible web interfaces.

## WCAG 2.2 Overview

- **Published**: October 2023
- **Principles**: POUR (Perceivable, Operable, Understandable, Robust)
- **Levels**: A (minimum), AA (standard target), AAA (enhanced)
- **Note**: SC 4.1.1 Parsing removed as obsolete

### New in WCAG 2.2

#### Level A (New)
| Criterion | Requirement |
|-----------|-------------|
| **2.5.7 Dragging Movements** | Provide single-pointer alternative to drag operations |
| **3.3.7 Redundant Entry** | Don't ask for same information twice in same session |

#### Level AA (New)
| Criterion | Requirement |
|-----------|-------------|
| **2.4.11 Focus Not Obscured (Min)** | Focused element at least partially visible |
| **2.5.8 Target Size (Minimum)** | Touch targets minimum 24x24 CSS pixels |
| **3.2.6 Consistent Help** | Help mechanism in same location across pages |
| **3.3.8 Accessible Auth (Min)** | No cognitive function tests for authentication |

#### Level AAA (New)
| Criterion | Requirement |
|-----------|-------------|
| **2.4.12 Focus Not Obscured (Enhanced)** | Entire focused element must be visible |
| **2.4.13 Focus Appearance** | Specific requirements for focus indicator contrast/size |
| **3.3.9 Accessible Auth (Enhanced)** | Stricter auth requirements |

## Complete A11y Project Checklist

### Content
- [ ] Plain language, 8th grade reading level (3.1.5)
- [ ] Unique, descriptive page titles (2.4.2)
- [ ] Button/link content unique and descriptive - no "click here" (1.3.1)
- [ ] Left-aligned text for LTR languages (1.4.8)

### Global Code
- [ ] Valid HTML (good practice)
- [ ] `lang` attribute on `<html>` (3.1.1)
- [ ] Unique `<title>` per page (2.4.2)
- [ ] Viewport zoom NOT disabled (1.4.4)
- [ ] Landmark elements (`nav`, `main`, `header`, `footer`) (4.1.2)
- [ ] Linear content flow - no non-standard tabindex (2.4.3)
- [ ] No `autofocus` attribute (2.4.3)
- [ ] Session timeouts can be extended (2.2.1)
- [ ] No `title` attribute tooltips except for iframes (4.1.2)

### Keyboard
- [ ] Visible focus style for all interactive elements (2.4.7)
- [ ] Focus order matches visual layout (1.3.2)
- [ ] No invisible focusable elements (2.4.3)
- [ ] Skip link visible when focused (2.4.1)
- [ ] No keyboard traps

### Images
- [ ] All `img` elements have `alt` attribute (1.1.1)
- [ ] Decorative images have `alt=""` (1.1.1)
- [ ] Complex images have extended descriptions (1.1.1)
- [ ] Images with text: include text in alt (1.1.1)

### Headings
- [ ] Headings introduce content sections (2.4.6)
- [ ] Only one `h1` per page/view (2.4.6)
- [ ] Logical heading sequence (2.4.6)
- [ ] No skipped heading levels (2.4.6)

### Lists
- [ ] Use `ol`, `ul`, `dl` for list content (1.3.1)

### Controls
- [ ] Use `<a>` for links with `href` (1.3.1)
- [ ] Links recognizable (underline or color+shape) (1.4.1)
- [ ] Controls have visible `:focus` states (2.4.7)
- [ ] Use `<button>` for buttons (1.3.1)
- [ ] Skip link provided (2.4.1)
- [ ] New tab/window links identified (G201)

### Tables
- [ ] Use `<table>` for tabular data (1.3.1)
- [ ] `<th>` with `scope="col"` or `scope="row"` (4.1.1)
- [ ] `<caption>` describes table content (2.4.6)

### Forms
- [ ] All inputs have associated `<label>` (3.2.2)
- [ ] Related inputs grouped with `<fieldset>`/`<legend>` (1.3.1)
- [ ] `autocomplete` where appropriate (1.3.5)
- [ ] Error list above form with links to fields (3.3.1)
- [ ] Errors connected via `aria-describedby` (3.3.1)
- [ ] States not communicated by color alone (1.4.1)

### Media
- [ ] No autoplay (1.4.2)
- [ ] Proper control markup (pressed states, sliders) (1.3.1)
- [ ] All media can be paused (2.1.1)

### Video
- [ ] Captions present (1.2.2)
- [ ] No seizure triggers (no strobing/flashing) (2.3.1)

### Audio
- [ ] Transcripts available (1.1.1)

### Appearance
- [ ] Works in High Contrast mode (1.4.1)
- [ ] Content readable at 200% text size (1.4.4)
- [ ] Good proximity maintained when zoomed (1.3.3)
- [ ] Information not conveyed by color alone (1.4.1)
- [ ] Instructions not visual/audio-only (1.3.3)
- [ ] Simple, consistent layout (1.4.10)

### Animation
- [ ] Subtle animations, no excessive flashing (2.3.1)
- [ ] Background video can be paused (2.2.2)
- [ ] Respects `prefers-reduced-motion` (2.3.3)

### Color Contrast (AA)
| Element | Minimum Ratio | Criterion |
|---------|---------------|-----------|
| Normal text | 4.5:1 | 1.4.3 |
| Large text (18pt+ or 14pt+ bold) | 3:1 | 1.4.3 |
| Icons | 3:1 | 1.4.11 |
| Input borders | 3:1 | 1.4.11 |
| Focus indicators | 3:1 | 1.4.11 |

### Mobile/Touch
- [ ] Works in any orientation (1.3.4)
- [ ] No horizontal scrolling (1.4.10)
- [ ] Touch targets minimum 44x44px (AAA) / 24x24px (AA)
- [ ] Sufficient space between interactive items (2.4.1)

## ARIA Rules

1. **First rule**: Don't use ARIA if native HTML works
2. **Roles**: Match element's actual purpose
3. **States**: Use `aria-pressed`, `aria-expanded`, `aria-selected` appropriately
4. **Labels**: Use `aria-label` or `aria-labelledby` when visible label absent
5. **Descriptions**: Use `aria-describedby` for additional context
6. **Live regions**: Use `aria-live` for dynamic content updates
7. **Hidden**: Use `aria-hidden="true"` for decorative elements

### Common ARIA Patterns

```html
<!-- Toggleable button -->
<button aria-pressed="false">Dark Mode</button>

<!-- Expandable section -->
<button aria-expanded="false" aria-controls="content">Menu</button>
<div id="content" hidden>...</div>

<!-- Tab panel -->
<div role="tablist">
  <button role="tab" aria-selected="true" aria-controls="panel1">Tab 1</button>
</div>
<div role="tabpanel" id="panel1">Content</div>

<!-- Live region for updates -->
<div aria-live="polite" aria-atomic="true">Status updates here</div>

<!-- Modal dialog -->
<div role="dialog" aria-labelledby="title" aria-modal="true">
  <h2 id="title">Dialog Title</h2>
</div>
```

## Focus Styles (Required)

```css
/* Visible focus indicator */
:focus-visible {
  outline: 2px solid var(--color-focus);
  outline-offset: 2px;
}

/* Remove outline only when not keyboard-focused */
:focus:not(:focus-visible) {
  outline: none;
}

/* High contrast mode support */
@media (forced-colors: active) {
  :focus-visible {
    outline: 3px solid CanvasText;
  }
}
```

## Reduced Motion Support (Required)

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```
